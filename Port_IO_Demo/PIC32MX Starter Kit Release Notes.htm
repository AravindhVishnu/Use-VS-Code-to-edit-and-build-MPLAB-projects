Overview

The PIC32MX General Purpose Starter Kit (DM320001) or PIC32MX USB Starter Board (DM320003) provide an out-of-box “hands-on” opportunity to program, debug and run a PIC32MX360F512L General Purpose or PIC32MX460F512L USB microcontroller using the included sample code projects,  MPLAB IDE (Integrated Development Environment), MPLAB C compiler for PIC32, PIC32 Peripheral Library and user Debug Print Library.  Refer to the PIC32MX Starter Kit User's Guide or PIC32MX USB Starter Board User's Guide for specific details and operation.

 

To see the latest PIC32MX Starter Kit available documentation, example code or other PIC32MX collateral, please visit www.microchip.com\pic32.

PIC32MX Starter Kit Technical Information

 

PIC32MX Dedicated Resources

LED1 (RED)        = PORTD.RD0

LED2 (YELLOW)     = PORTD.RD1

LED3 (GREEN)      = PORTD.RD2

SW1               = PORTD.RD6             (change notice pin CN15)

SW2               = PORTD.RD7             (change notice pin CN16)

SW3               = PORTD.RD13           (change notice pin CN19)

JTAG.TMS          = PORTA.RA0             (used by debugger)

JTAG.TCK          = PORTA.RA1             (used by debugger)

JTAG.TDO          = PORTA.RA5             (used by debugger)

JTAG.TDI          = PORTA.RA4             (used by debugger)

PGC2              = PORTB.RB6             (used by debugger)

PGD2              = PORTB.RB7             (used by debugger)

 

Note:  There are no pullup resistors on SW1, SW2 or SW3.  To enable the PORTD pullups for SW1, SW2 and SW3, use the following:
CNPUE = (CN15_PULLUP_ENABLE | CN16_PULLUP_ENABLE | CN19_PULLUP_ENABLE);

 


PIC32MX Device Configuration Fuses

The MPLAB C compiler for PIC32 supports #pragma directives to allow PIC32MX device configuration fuse bits to be specified in source code.  These are shown below. To see all the PIC32MX family device fuse configurations, refer to the MPLAB IDE menu HelpTopics>PIC32MXConfigSettings.

#pragma config FNOSC = PRIPLL       // Oscillator Selection
#pragma config FPLLIDIV = DIV_2     // PLL Input Divider (PIC32MX Starter Kit: use divide by 2 only)
#pragma config FPLLMUL = MUL_20     // PLL Multiplier
#pragma config FPLLODIV = DIV_1     // PLL Output Divider
#pragma config FPBDIV = DIV_1       // Peripheral Clock divisor
#pragma config FWDTEN = OFF         // Watchdog Timer
#pragma config WDTPS = PS1          // Watchdog Timer Postscale
#pragma config FCKSM = CSDCMD       // Clock Switching & Fail Safe Clock Monitor
#pragma config OSCIOFNC = OFF       // CLKO Enable
#pragma config POSCMOD = HS         // Primary Oscillator
#pragma config IESO = OFF           // Internal/External Switch-over
#pragma config FSOSCEN = OFF        // Secondary Oscillator Enable
#pragma config CP = OFF             // Code Protect
#pragma config BWP = OFF            // Boot Flash Write Protect
#pragma config PWP = OFF            // Program Flash Write Protect
#pragma config ICESEL = ICS_PGx2    // ICE/ICD Comm Channel Select
#pragma config DEBUG = OFF          // Debugger Disabled for Starter Kit

Note: To set the PIC32MX device configuration fuse bits from the MPLAB IDE main menu, select Configure>Configuration Bits and un-check the box "Configuration Bits set in code".  Configure the desired fuse bits for the application.  Caution - Setting the device configuration bits to cause the PIC32MX device to operate faster than the maximum 80MHz system clock speed may cause the device to stop communicating with the on-board PIC18F4550 Starter Kit debugger. See the IMPORTANT notice below regarding system clock oscillator.

 


Important PIC32MX Starter Kit Tips

JTAG must be enabled when using the Starter Kit
The 18F4550 on-board debugger uses JTAG to communicate with the PIC32MX device. The PIC32MX DDPCON register contains a bit, DDPCON<3>, which enables or disables the PIC32MX JTAG port.  On a device reset, this bit is set = 1 by default, enabling JTAG. This bit is configurable during runtime and if it is mistakenly set = 0, the on-board debugger can no longer talk to the PIC32MX device and will hang. It is important to leave the PIC32MX JTAG enabled, DDPCON<3> = 1, for proper operation of the PIC32MX Starter Kit.

When a Data Breakpoint is reached, press "step" once before pressing "Run"
MPLAB has the ability to cause a debugger break when a memory location or peripheral register is accessed. When a data breakpoint causes MPLAB to halt program execution it appears that the debugger is "stuck" on the data breakpoint when "Run" is pressed. To continue running the program must be single-stepped first before pressing "Run".

Leave the USB cable attached while debugging
The PIC32MX Starter Kit does not reconnect automatically when the USB cable is removed and then re-attached. If the USB cable is removed, it is necessary to perform the following actions:
A)  From the Debug menu, select "Debugger -> None"
B)  Connect USB cable
C)  Select "Debugger -> PIC32MX Starter Kit"

Some Configuration-Bit settings are not supported in this release
In this release, the starter kit does not support Code-Protect features of the PIC32MX processor. Also not supported is debugging while running with the 32KHz oscillator.
See sk_erase.exe utility tech tip below. 

Do not set System Clock Oscillator PLL fuse bits for frequency greater than 80MHz
Communication with PIC18F4550 Starter Kit Debugger may stop if the oscillator PLL fuse bits are configured for system clock frequencies greater than 80MHz.
See sk_erase.exe utility tech tip below.

PIC32MX Device Re-Flash Utility (sk_erase.exe)

If the PIC32MX device configuration bits are mistakenly written to a non-supported state, run the sk_erase.exe utility located in <install dir>\tools to re-flash the PIC32MX with a factory default device configuration.  Be sure to connect the USB cable to the PIC32MX debug mini connector (connector next to DEBUG LED) before launching the sk_erase utility.

PIC32 Peripheral Library support

The PIC32 Peripheral Library macros and functions are designed to simplify PIC32MX peripheral configuration and control. These peripheral libraries are part of the MPLAB C for PIC32 compiler installation.

·        To use any PIC32 Peripheral Library macro or function, add the following to your source code file: "#include <plib.h>"

·        If not using the PIC32 Peripheral Library but directly addressing any peripheral SFR, add the following to your source code file: "#include <p32xxxx.h>"


PIC32 Peripheral Library Device Performance macros


The PIC32 Peripheral Library provides macros and functions to easily configure the PIC32MX device parameters for optimal system performance.  These functions will properly set the required system parameters, such as number of Flash wait states and RAM wait states, based on the system clock frequency argument passed to the function, as well as enabling the CPU instruction cache.  In addition to setting the device parameters mentioned, these functions will also set the peripheral bus clock (PBCLK) frequency based on which function is used.  These two functions are SYSTEMConfigPerformance( ) and SYSTEMConfig( ).

SYSTEMConfigPerformance( ) ignores the PBCLK fuse selection FPBDIV (set either in code or from the MPLAB IDE menu) and calculates the fastest possible PBCLK based on the system clock frequency parameter passed in the function call. Typically, this is a PBCLK divisor = 1:1, which means the PBCLK will operate at the same frequency as the system clock (SYSCLK).

 

·        Example: SYSTEMConfigPerformance(80000000);

 


SYSTEMConfig( ) sets the PBCLK frequency defined by the FPBDIV configuration fuse set in code or from the MPLAB IDE Configuration Bits menu. Use this function if you want to maintain the PBCLK frequency set by the FPBDIV fuse.

 

·        Example: SYSTEMConfig(80000000, SYS_CFG_WAIT_STATES | SYS_CFG_PCACHE);

 

 

Debug Print Library support

Important - The Debug Print Library macros are supported in DEBUG mode only.

 

The Debug Print Library macros DBINIT, DBPUTC, DBPUTS, DPPRINTF, DBGETS and DBGETC allow the user to add basic input/output capability between a PIC32 Starter Kit application and the MPLAB IDE Starter Kit console window during runtime debugging.  These Debug Print Library macros are defined in “db_utils.h”, located in <install dir>\microchip\include.  They provide the interface for all of the Debug Print Library functions in “db_utils.a” with the exception of DBPRINTF macro which defines the interface to the printf( ) function provided in the MPLAB C Compiler for PIC32. The db_utils.a library file is located in <install dir>\microchip\common.

 

The sample projects provided with the PIC32MX Starter Kit have been properly configured to the use the Debug Print Library macros.

 

To use the Debug Print Library utility macros in your own application, do the following:

1.    Add "#include "db_utils.h" to your source code and either
a) specify the path to db_utils.h in the MPLAB IDE menu Project Build Options>Directories>Include Search Path
b) or place a copy of the db_utils.h file in your project directory.

2.    Add the db_utils.a file to the library branch of your project tree.

3.    Add the macro definition "PIC32_STARTER_KIT" to the compiler options specified in the MPLAB IDE menu Project Build Options>C Compiler>Macro Definitions (preferred method).  This creates a global definition for the entire project, which means Debug Print Macros can be used in multiple source files in the project.  An alternative is to simply add the “#define PIC32_STARTER_KIT” macro definition directly in your source code file where the Debug Print macros will be used.  Important - Be sure the place the “#define PIC32_STARTER_KIT” before the line containing “#include db_utils.h” or a compiler error will be generated.

4.    Finally, be sure to call DBINIT( ) first before any other Debug Print Library macro.

 

Using DBPRINTF( )

 

The DBPRINTF macro allows formatted string output to be specified, which permits variables and register contents to be output to the MPLAB Starter Kit console window during debugging.

 

For example, the following line of code: DBPRINTF("The value of OSCCON register is %X \n",OSCCON);

 

produces this output in the MPLAB Starter Kit console window: [TargetOUT] The value of OSCCON register is 391F7702

 


Floating Point numbers used with DBPRINTF( )

If floating point numbers are not passed to the printf( ) function, the amount of FLASH memory required by the application can be reduced by linking without the floating point libraries.  This is can be achieved by using size (–Os) optimization and selecting “Exclude floating-point library” (-mno-float) options in the MPLAB PIC32 Linker script Build Options menu.  Refer to the compiler user’s manual for further details on printf( ) and compiler options.


NOTE: All PIC32MX Starter Kit tutorial code example projects are not linked with floating point libraries.  If you want to use floating point numbers with the printf( ) function in these example projects, you will need to change the linker script build options to include floating-point libraries (see above).

PIC32MX Starter Kit FAQs

  1. The debugger has stopped communicating with MPLAB IDE after programming the PIC32 Starter Kit –The PIC32 configuration fuses may have been programmed with either an unsupported mode or the Oscillator PLL fuses were programmed to generate a system clock faster than 80MHz.  See the sk_erase.exe topic above to re-flash the default PIC32 settings.

  2. Is it possible to program a PIC32MX Starter Kit application to run in “Release” mode? – Yes.  Select RELEASE from the MPLAB IDE drop down menu in the main tool bar and re-compile the project. Program the device.  MPLAB will warn of an “Image Platform Mismatch” and prompt you to rebuild the project.  Politely respond NO and proceed with the programming. When the programming is complete, select debugger = none in the MPLAB IDE menu (deselecting the debugger before disconnecting is always recommended), disconnect the USB cable to remove power from the PIC32, then re-connect the USB cable to apply power and the application will now be running in RELEASE mode.

  3. No debug print messages appear in the MPLAB IDE Starter Kit Window when using DBPUTS or DBPRINTF macros.  - Be sure to add the macro definition "PIC32_STARTER_KIT" in the MPLAB IDE menu Project Build Options>C Compiler>Macro Definitions and be sure the project is built for DEBUG, not RELEASE mode.
